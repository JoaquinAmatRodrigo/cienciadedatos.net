Introducción¶


Una serie temporal (time series) es una sucesi칩n de datos ordenados cronol칩gicamente, espaciados a intervalos iguales o desiguales. El proceso de forecasting consiste en predecir el valor futuro de una serie temporal, bien modelando la serie 칰nicamente en funci칩n de su comportamiento pasado (autorregresivo) o empleando otras variables externas.

En este documento se muestra un ejemplo de c칩mo utilizar m칠todos machine learning y modelado estad칤stico (ARIMA) para predecir el n칰mero de visitas diarias que recibe una p치gina web.

Caso de uso¶


Se dispone del historial de visitas diarias a la web cienciadedatos.net desde el 01/07/2020. Se pretende generar un modelo de forecasting capaz de predecir el tr치fico web que tendr치 la p치gina a 7 d칤as vista. En concreto, el usuario quiere ser capaz de ejecutar el modelo cada lunes y obtener las predicciones de tr치fico diario hasta el lunes siguiente.

Con el objetivo de poder evaluar de forma robusta la capacidad del modelo acorde al uso que se le quiere dar, conviene no limitarse a predecir 칰nicamente los 칰ltimos 7 d칤as de la serie temporal, sino simular el proceso completo. Dos de las estrategias que se pueden emplear a la hora de realizar este tipo de validaci칩n son:

Time series cross-validation

El modelo se entrena cada vez antes de realizar las predicciones, de esta forma, se incorpora toda la informaci칩n disponible hasta el momento. Se trata de una adaptaci칩n del proceso de cross-validation en el que, en lugar de hacer un reparto aleatorio de las observaciones, el conjunto de entrenamiento se incrementa de forma secuencial, manteniendo el orden temporal de los datos.

Diagrama de time series cross-validation con un tama침o inicial de entrenamiento de 10 observaciones y un horizonte de predicci칩n de 3 steps.

Backtesting

Es una estrategia similar a la de time series cross-validation pero sin reentrenar el modelo. El modelo se entrena una 칰nica vez con un conjunto inicial y con 칠l se realizan las predicciones de forma secuencial sin actualizar el modelo y siguiendo el orden temporal de los datos. Esta estrategia tiene la ventaja de ser mucho m치s r치pida que time series cross-validation puesto que el modelo solo se entrena una vez. La desventaja es que el modelo no incorpora la 칰ltima informaci칩n disponible por lo que puede perder capacidad predictiva con el tiempo.


Diagrama de time series backtesting con un tama침o inicial de entrenamiento de 10 observaciones y un horizonte de predicci칩n de 3 steps.

El m칠todo de validaci칩n m치s adecuada depender치 de cu치l sea la estrategia seguida en la puesta en producci칩n, en concreto, de si el modelo se va a reentrenar peri칩dicamente o no antes de que se ejecute el proceso de predicci칩n.
