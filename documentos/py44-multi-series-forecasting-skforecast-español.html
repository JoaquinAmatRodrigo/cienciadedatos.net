Forecasting multiseries¶

En el forecasting de series temporales univariantes, una Ãºnica serie temporal se modela como una combinaciÃ³n lineal o no lineal de sus lags. Es decir, los valores pasados de la serie se utilizan para predecir su comportamiento futuro. En el forecasting multiserie, dos o mÃ¡s series temporales se modelan conjuntamente mediante un Ãºnico modelo. Se pueden distinguir dos estrategias:

MÃºltiples series temporales no multivariantes

En esta situaciÃ³n, cada serie temporal es independiente de las demÃ¡s o, dicho de otro modo, los valores pasados de una serie no se utilizan como predictores de las otras series. Â¿Por quÃ© es Ãºtil entonces modelar todo junto? Aunque las series no dependen unas de otras, pueden seguir el mismo patrÃ³n intrÃ­nseco en cuanto a sus valores pasados y futuros. Por ejemplo, en una misma tienda, las ventas de los productos A y B pueden no estar relacionadas, pero siguen la misma dinÃ¡mica, la de la tienda.

Para predecir los siguientes n steps, se sigue una estrategia recurisva, recursive multi-step forecasting.

MÃºltiples series temporales multivariantes

Todas las series se modelan teniendo en cuenta que cada serie temporal depende no sÃ³lo de sus valores pasados, sino tambiÃ©n de los valores pasados de las demÃ¡s series. Se espera que el modelo no sÃ³lo aprenda la informaciÃ³n de cada serie por separado, sino que tambiÃ©n las relacione. Por ejemplo, las mediciones realizadas por todos los sensores (caudal, temperatura, presiÃ³n...) instalados en una mÃ¡quina industrial como un compresor. Series temporales multivariantes user guide.

  Note

La clase ForecasterAutoregMultiSeries cubre el caso de uso de series temporales no multivariantes. API Reference

  Note

Para aplicar forecasting a mÃºltiples series temporales no multivariantes con predictores custom se puede usar la clase ForecasterAutoregMultiSeriesCustom. GuÃ­a de usuario: Custom predictors for recursive multi-step forecasting.

Ventajas y limitaciones¶


El forecasting multiserie no siempre superan al modelado de la serie individual. CuÃ¡l de ellas funciona mejor depende en gran medida de las caracterÃ­sticas del caso de uso al que se aplican. Sin embargo, conviene tener en cuenta la siguiente heurÃ­stica:

Ventajas de los modelos multiseries:

  • Es mÃ¡s fÃ¡cil mantener y controlar un solo modelo que varios.

  • Dado que todas las series temporales se combinan durante el entrenamiento, cuando las series sean cortas (pocos datos) el modelo tendrÃ¡ una mayor capacidad de aprendizaje al disponer de mÃ¡s observaciones.

  • Al combinar mÃºltiples series temporales, el modelo puede aprender patrones mÃ¡s generalizables.

Desventajas de los modelos multiseries:

  • Si las series no siguen la misma dinÃ¡mica interna, el modelo puede aprender un patrÃ³n que no represente a ninguna de ellas.

  • Las series pueden enmascararse unas a otras, por lo que el modelo puede no predecirlas todas con el mismo rendimiento.

  • Es mÃ¡s exigente desde el punto de vista computacional (tiempo y recursos) entrenar y realizar backtesting de un modelo grande que de varios pequeÃ±os.

Caso de uso¶


El objetivo de este estudio es comparar los resultados obtenidos por un modelo multiserie frente a la utilizaciÃ³n de un modelo diferente para cada serie.

Los datos se han obtenido de Store Item Demand Forecasting Challenge. Este dataset contiene 913.000 transacciones de ventas desde el 2013-01-01 hasta el 2017-12-31 para 50 productos (SKU) en 10 tiendas. El objetivo es predecir las ventas de los prÃ³ximos 7 dÃ­as de 50 artÃ­culos diferentes en una tienda utilizando el historial disponible de 5 aÃ±os.

Impacto de la longitud de la serie¶



Si una serie temporal tiene pocas observaciones, la cantidad de informaciÃ³n disponible para que el modelo aprenda es limitada. Este es un problema comÃºn en casos reales en los que no hay muchos datos histÃ³ricos disponibles. Los forecasters multiseries no multivariantes combinan todas las series durante el entrenamiento, por lo que pueden acceder a mÃ¡s datos.


En esta secciÃ³n, se realiza la misma comparaciÃ³n entre el forecasting de una sola serie y el forecasting multiserie, pero, esta vez, la longitud de las series es mucho menor.

Pesos en forecasting multiseries¶



Los pesos se utilizan para controlar la influencia que tiene cada observaciÃ³n en el entrenamiento del modelo. ForecasterAutoregMultiseries acepta dos tipos de pesos:


    

  • series_weights controla la importancia relativa de cada serie. Si una serie tiene el doble de peso que las demÃ¡s, las observaciones de esa serie influyen el doble en el entrenamiento. Cuanto mayor sea el peso de una serie en relaciÃ³n con las demÃ¡s, mÃ¡s se centrarÃ¡ el modelo en intentar aprender esa serie.
    

    • 

  • weight_func controla la importancia relativa de cada observaciÃ³n en funciÃ³n del Ã­ndice. Por ejemplo, una funciÃ³n que asigna un peso menor a ciertas fechas.
    

    • 



Si se indican los dos tipos de pesos, estos se multiplican para crear los pesos finales como se muestra en la figura. El sample_weight resultante no puede contener valores negativos.


MÃ¡s informaciÃ³n sobre weights in multi-series forecasting y weighted time series forecasting con skforecast.


En este ejemplo, item_1 tiene una mayor importancia relativa entre series (pesa 3 veces mÃ¡s que el resto de series), y las observaciones entre '2013-12-01' y '2014-01-31' se consideran no representativas y se les aplica un peso de 0.

# ======================================================================================

# `series_weights`
# ================
series_weights = {'item_1': 3.} # Las series que no aparezcan en el dict tienen un peso de 1

# `weight_func`
# ================
# Se puede pasar un dict a `weight_func` para aplicar diferentes funciones para cada serie
# Las series que no aparezcan en el dict tienen un peso de 1
def custom_weights(index):
    """
    Devuelve 0 si el Ã­ndice estÃ¡ entre '2013-12-01' y '2014-01-31', 1 en caso contrario.
    """
    weights = np.where(
                  (index >= '2013-12-01') & (index <= '2014-01-31'),
                   0,
                   1
              )
    
    return weights

                 transformer_exog   = None,
