Introducción¶

Los modelos basados en rboles, incluidos los rboles de decisin, random forests y gradient boosting machines (GBMs), son conocidos por su eficacia y su uso generalizado en multitud de aplicaciones de machine learning. Sin embargo, tienen limitaciones cuando se trata de extrapolar, es decir, hacer predicciones o estimaciones ms all del rango de los datos observado. Esta limitacin resulta especialmente crtica cuando se trata de modelar series temporales que tienen tendencia. Debido a que estos modelos carecen de la capacidad para predecir valores ms all del rango observado durante el entrenamiento, sus valores pronosticados se desviarn de la tendencia subyacente.

Se han propuesto varias estrategias para hacer frente a este reto, y uno de los enfoques ms comunes es el uso de la diferenciacin. El proceso de diferenciacin consiste en calcular las diferencias entre observaciones consecutivas de la serie temporal. En lugar de modelizar directamente los valores absolutos, la atencin se centra en modelizar el ratio de cambio relativo. Una vez estimadas las predicciones, puede invertirse la transformacin para obtener los valores en la escala original.

La librera skforecast, versin 0.10.0 o superior, introduce un nuevo argumento differentiation dentro de sus Forecasters para indicar que se debe aplicar un proceso de diferenciacin antes de entrenar el modelo. Esto se consigue utilizando internamente una nueva clase llamada skforecast.preprocessing.TimeSeriesDifferentiator. Cabe destacar que todo el proceso de diferenciacin se ha automatizado y sus efectos se invierten durante la fase de prediccin. Esto garantiza que los valores predichos se encuentren en la misma escala que los datos originales.

Este documento muestra cmo la diferenciacin permite modelizar series temporales con tendencia positiva utilizando modelos basados en rboles (random forest y gradient boosting xgboost).

Forecasting con Random Forest y Gradient Boosting¶

Se crean dos forecasters, uno con un regresor RandomForestRegressor de scikit-learn y otro con XGBoost. Ambos se entrenan con datos de 1949-01-01 a 1956-01-01 y generan predicciones para los prximos 60 meses (5 aos).

Detalles de la diferenciación de series temporales¶

El ejemplo anterior muestra lo fcil que es introducir la diferenciacin en el proceso de forecasting gracias a las funcionalidades disponibles en skforecast. Sin embargo, hay que aplicar varias transformaciones no triviales para conseguir una interaccin fluida.

En los siguientes apartados, se presenta la clase TimeSeriesDifferentiator y se muestran sus principales ventajas.

  • La clase es capaz de diferenciaciar e integracin (diferenciacin inversa) cualquier serie temporal.

  • Explicacin de por qu la gestin interna de la diferenciacin tiene ventajas sobre el enfoque tradicional de pre-transformar toda la serie temporal antes de iniciar el entrenamiento del modelo.

  • Cmo gestionar la diferenciacin cuando se aplica el Forecaster a nuevos datos que no siguen inmediatamente a los datos de entrenamiento).

TimeSeriesDifferentiator¶

TimeSeriesDifferentiator es un clase que sigue la API de sklearn de preprocesamiento. Esto significa que tiene el mtodo fit, transform, fit_transform y inverse_transform.

In [7]:
# Diferenciacin con TimeSeriesDifferentiator
# ==============================================================================
y = np.array([5, 8, 12, 10, 14, 17, 21, 19], dtype=float)
diferenciador = TimeSeriesDifferentiator()
diferenciador.fit(y)
y_diff = diferenciador.transform(y)

print(f"Serie original    : {y}")
print(f"Serie diferenciada: {y_diff}")

Serie original    : [ 5.  8. 12. 10. 14. 17. 21. 19.]
Serie diferenciada: [nan  3.  4. -2.  4.  3.  4. -2.]

La diferenciacin puede revertirse (integracin) con el mtodo inverse_transform, recuperando as la serie original.

In [8]:
# Revertir transformacin
# ==============================================================================
diferenciador.inverse_transform(y_diff)
Out[8]:
array([ 5.,  8., 12., 10., 14., 17., 21., 19.])

⚠ Warning

El proceso de transformacin inversa inverse_transform es aplicable exclusivamente a las mismas series temporales que han sido previamente diferenciadas utilizando el mismo objeto TimeSeriesDifferentiator. Esta restriccin surge de la necesidad de utilizar los n valores iniciales de la serie temporal (n igual al orden de diferenciacin) para revertir efectivamente la diferenciacin. Estos valores se capturan durante la invocacin del mtodo fit.


✎ Nota

Un mtodo adicional inverse_transform_next_window est disponible en la clase TimeSeriesDifferentiator. Este mtodo est pensado para ser utilizado dentro de los Forecasters para invertir la diferenciacin de los valores predichos. Si el regresor dentro del Forecaster est entrenado con una serie temporal diferenciada, entonces sus predicciones tambin lo estarn. El mtodo inverse_transform_next_window permite devolver las predicciones a la escala original, suponiendo que siguen inmediatamente despus de los ltimos valores observados (last_window).

Diferenciación interna frente a preprocesamiento¶

Los Forecasters gestionan internamente el proceso de diferenciacin, por lo que no es necesario preprocesar las series temporales ni postprocesar sus predicciones. Esto tiene varias ventajas, pero antes de entrar en ellas, se comparan los resultados de ambos enfoques.

A continuacin, los resultados del proceso de backtesting se someten a un anlisis comparativo. Esta comparacin es ms compleja que la anterior, ya que el proceso de deshacer la diferenciacin debe realizarse por separado para cada particin del backtesting.

In [12]:
# Backtesting con la serie temporal diferenciada en el preprocesamiento
# ==============================================================================
forecaster_1 = ForecasterRecursive(
                   regressor = RandomForestRegressor(random_state=963),
                   lags      = 15
               )
cv = TimeSeriesFold(
        steps              = 5,
        initial_train_size = len(datos_diff.loc[:end_train]),
        refit              = True,
        fixed_train_size   = False,
     )
_, predicciones_1 = backtesting_forecaster(
                        forecaster    = forecaster_1,
                        y             = datos_diff,
                        cv            = cv,
                        metric        = 'mean_squared_error',
                        n_jobs        = 'auto',
                        verbose       = False,
                        show_progress = True  
                    )

# Invertir diferenciacin para obtener predicciones en la escala original. Las
# predicciones de cada particin deben revertirse individualmente. Un id se aade
# a cada prediccin para identificar la particin a la que pertenece.
predicciones_1 = predicciones_1.rename(columns={'pred': 'pred_diff'})
folds = len(predicciones_1) / cv.steps
folds = int(np.ceil(folds))
predicciones_1['backtesting_fold_id'] = np.repeat(range(folds), cv.steps)[:len(predicciones_1)]

# Sumar el valor observado anteriormente (solo a la primera prediccin de cada particin)
valores_anteriores = datos.shift(1).loc[predicciones_1.index].iloc[::cv.steps]
valores_anteriores.name = 'valor_anterior'
predicciones_1 = predicciones_1.merge(
                    valores_anteriores,
                    left_index  = True,
                    right_index = True,
                    how         = 'left'
                )
predicciones_1 = predicciones_1.fillna(0)
predicciones_1['suma_valores'] = (
    predicciones_1['pred_diff'] + predicciones_1['valor_anterior']
)

# Revert differentiation using the cumulative sum by fold
predicciones_1['pred'] = (
    predicciones_1
    .groupby('backtesting_fold_id')
    .apply(lambda x: x['suma_valores'].cumsum(), include_groups=False)
    .to_numpy()
)

Como se ha demostrado, los valores son equivalentes cuando se diferencian las series temporales con un paso de preprocesamiento o cuando se permite que el Forecaster gestione la diferenciacin internamente, por qu la segunda alternativa es mejor?

  • Al gestionar el Forecaster internamente todas las transformaciones garantiza que se apliquen las mismas transformaciones cuando el modelo se ejecuta con nuevos datos.

  • Cuando el modelo se aplica a nuevos datos que no siguen inmediatamente a los datos de entrenamiento (por ejemplo, si un modelo no se vuelve a entrenar antes de cada fase de prediccin), el Forecaster aumenta automticamente el tamao de la ltima ventana (last_window) necesaria para generar los predictores, adems de aplicar la diferenciacin a los datos entrantes e invertirla en las predicciones finales.

Estas transformaciones no son triviales y son muy propensas a errores, por lo que skforecast intenta evitar complicar en exceso la ya de por s difcil tarea de predecir series temporales.
