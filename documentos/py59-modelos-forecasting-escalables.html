Introduction¶

En escenarios que implican la predicciÃ³n de cientos o miles de series temporales, surge una decisiÃ³n crucial: Â¿se deben desarrollar modelos individuales para cada serie o se debe utilizar un Ãºnico modelo para manejarlas todas a la vez?

En la modelaciÃ³n de una sola serie (modelo de forecasting local), se crea un modelo de predicciÃ³n independiente para cada serie temporal. Aunque este mÃ©todo proporciona una comprensiÃ³n exhaustiva de cada serie, su escalabilidad puede verse dificultada por la necesidad de crear y mantener cientos o miles de modelos.

La modelizaciÃ³n multiserie (modelo de forecasting global) consiste en crear un Ãºnico modelo predictivo que tenga en cuenta todas las series temporales simultÃ¡neamente. Este enfoque es eficiente desde el punto de vista computacional, fÃ¡cil de mantener y puede producir generalizaciones mÃ¡s sÃ³lidas, aunque potencialmente a costa de sacrificar algunos conocimientos individuales.

Variables exógenas¶

Variables categóricas¶

LightGBM permite incluir variables categÃ³ricas en el modelo sin necesidad de preprocesamiento. Para permitir la detecciÃ³n automÃ¡tica de variables categÃ³ricas en un Forecaster, primero las variables categÃ³ricas deben codificarse como enteros (codificaciÃ³n ordinal) y luego almacenarse como tipo category. Esto es necesario porque skforecast utiliza una matriz numÃ©rica internamente para acelerar el cÃ¡lculo, y LightGBM requiere que las caracterÃ­sticas categÃ³ricas estÃ©n codificadas como category para ser detectadas automÃ¡ticamente. TambiÃ©n es necesario establecer el parÃ¡metro categorical_features en 'auto' durante la inicializaciÃ³n del modelo de forecasting utilizando fit_kwargs = {'categorical_feature': 'auto'}.

⚠ Warning

Las cuatro principales implementaciones de gradient boosting - LightGBM, scikit-learn's HistogramGradientBoosting, XGBoost y CatBoost - son capaces de manejar directamente las variables categÃ³ricas dentro del modelo. Sin embargo, es importante tener en cuenta que cada implementaciÃ³n tiene sus propias configuraciones, beneficios y posibles problemas. Para comprender completamente cÃ³mo utilizar estas implementaciones, se recomienda consultar la guÃ­a del usuario de skforecast para una comprensiÃ³n detallada.

ForecasterRecursiveMultiSeries permite modelar series temporales de diferentes longitudes y utilizando distintas variables exÃ³genas. Cuando las series tienen longitudes diferentes, los datos deben transformarse en un diccionario. Las claves del diccionario son los nombres de las series y los valores son las propias series. Para ello, se utiliza la funciÃ³n series_long_to_dict, que toma un DataFrame en Â«formato largoÂ» y devuelve un diccionario de Series de Pandas. Del mismo modo, cuando las variables exÃ³genas son diferentes (valores o variables) para cada serie, los datos deben transformarse en un diccionario. Las claves del diccionario son los nombres de las series y los valores son las propias variables exÃ³genas. Se utiliza la funciÃ³n exog_long_to_dict, que toma el DataFrame en Â«formato largoÂ» y devuelve un diccionario de variables exÃ³genas (series de Pandas o DataFrames de Pandas).


Cuando todas las series tienen la misma longitud y las mismas variables exÃ³genas, no es necesario utilizar diccionarios. Las series se pueden pasar como un Ãºnico DataFrame con cada serie en una columna, y las variables exÃ³genas se pueden pasar como un DataFrame con la misma longitud que la serie.

Búsqueda de hiperparámetros¶














La bÃºsqueda de hiperparÃ¡metros y lags implica probar sistemÃ¡ticamente diferentes valores de hiperparÃ¡metros (y/o lags) para encontrar la configuraciÃ³n Ã³ptima que ofrezca el mejor rendimiento. skforecast proporciona dos mÃ©todos diferentes para evaluar cada configuraciÃ³n candidata:


    

  • Backtesting: en este mÃ©todo, el modelo predice varios pasos a futuro en cada iteraciÃ³n, utilizando el mismo horizonte de predicciÃ³n y la misma estrategia de reentrenamiento que se utilizarÃ­an si se desplegara el modelo. De este modo, se simula un escenario de predicciÃ³n real en el que el modelo se reentrena y actualiza a lo largo del tiempo.
    

    • 

  • One-Step Ahead: EvalÃºa el modelo utilizando solo predicciones de un paso a futuro. Este mÃ©todo es mÃ¡s rÃ¡pido porque requiere menos iteraciones, pero solo evalua el rendimiento del modelo en el siguiente paso temporal (t+1).
    

    • 



Cada mÃ©todo utiliza una estrategia de evaluaciÃ³n diferente, por lo que pueden producir resultados distintos. Sin embargo, a largo plazo, se espera que ambos mÃ©todos converjan a selecciones similares de hiperparÃ¡metros Ã³ptimos. El mÃ©todo de One-Step Ahead es mucho mÃ¡s rÃ¡pido que el backtesting porque requiere menos iteraciones, pero solo prueba el rendimiento del modelo en el siguiente instante de tiempo. Se recomienda realizar un backtest del modelo final para obtener una estimaciÃ³n mÃ¡s precisa del rendimiento cuando se predicen varios pasos a futuro.

Selección de predictores¶














La selecciÃ³n de predictores es el proceso de seleccionar un subconjunto de predictores relevantes (variables) para su uso en la construcciÃ³n del modelo. Las tÃ©cnicas de selecciÃ³n de predictores se utilizan por varias razones: para simplificar los modelos y hacerlos mÃ¡s fÃ¡ciles de interpretar, para reducir el tiempo de entrenamiento, para evitar los problemas de dimensionalidad, para mejorar la generalizaciÃ³n reduciendo el sobreajuste (formalmente, la reducciÃ³n de la varianza), entre otros.


Skforecast es compatible con los mÃ©todos de selecciÃ³n implementados en scikit-learn. Existen varios mÃ©todos de selecciÃ³n de caracterÃ­sticas, pero los mÃ¡s comunes son:


    

  • Recursive feature elimination (RFE)
    

    • 

  • Sequential Feature Selection (SFS)
    

    • 

  • Feature selection based on threshold (SelectFromModel)
    

    • 




















    
    💡 Tip



La selecciÃ³n de predictores es una herramienta poderosa para mejorar el rendimiento de los modelos de machine learning. Sin embargo, es computacionalmente costosa y puede llevar tiempo. Dado que el objetivo es encontrar el mejor subconjunto de variables, no el mejor modelo, no es necesario utilizar todo el conjunto de datos o un modelo muy complejo. En su lugar, se recomienda utilizar un pequeÃ±o subconjunto de datos y un modelo simple. Una vez que se haya identificado el mejor subconjunto de variables, el modelo puede entrenarse utilizando todo el conjunto de datos y una configuraciÃ³n mÃ¡s compleja.

Clustering series temporales¶














La idea que hay detrÃ¡s de modelar varias series al mismo tiempo es poder capturar los patrones principales que rigen dichas series, reduciendo asÃ­ el impacto del ruido que pueda haber en cada una de ellas. Esto significa que las series que se comportan de manera similar pueden beneficiarse de ser modelizadas juntas. Una forma de identificar posibles grupos de series es realizar un estudio de +cluatering antes de modelizarlas. Si como resultado del clustering* se identifican grupos claros, es apropiado modelar cada uno de ellos por separado.


El clustering es una tÃ©cnica de anÃ¡lisis no supervisado que agrupa un conjunto de observaciones en clÃºsteres que contienen observaciones consideradas homogÃ©neas, mientras que las observaciones en diferentes clÃºsteres se consideran heterogÃ©neas. Los algoritmos que agrupan series temporales se pueden dividir en dos grupos: aquellos que utilizan una transformaciÃ³n para crear variables antes de agrupar (clustering de series temporales basado en caracterÃ­sticas) y aquellos que trabajan directamente en las series temporales (medidas de distancia elÃ¡stica).


    

  • Clustering basado en caracterÃ­sticas de series temporales: se extraen variables que describen las caracterÃ­sticas estructurales de cada serie temporal y luego se introducen en algoritmos de clustering. Estas variables se obtienen aplicando operaciones estadÃ­sticas que capturan mejor las caracterÃ­sticas subyacentes: tendencia, estacionalidad, periodicidad, correlaciÃ³n serial, asimetrÃ­a, curtosis, caos, no linealidad y auto-similitud.
    

    • 

  • Medidas de distancia elÃ¡stica: este enfoque trabaja directamente en las series temporales, ajustando o Â«reajustandoÂ» las series en comparaciÃ³n con otras. La medida mÃ¡s conocida de esta familia es el Dynamic Time Warping (DTW).
    

    • 



Para un ejemplo detallado de cÃ³mo el clustering de series temporales puede mejorar los modelos de forecasting, consulte Modelos de forecasting globales: AnÃ¡lisis comparativo de modelos de una y mÃºltiples series.
