Series temporales con valores faltantes¶

En muchos casos reales de forecasting, aunque se disponga de datos histÃ³ricos, es habitual que las series temporales estÃ©n incompletas. La presencia de valores faltantes (missing) en los datos es un problema importante, ya que la mayorÃ­a de los algoritmos de forecasting requieren que las series temporales estÃ©n completas para poder entrenar un modelo.

Una estrategia comÃºnmente empleada para evitar este problema consiste en imputar los valores que faltan antes de entrenar el modelo, por ejemplo, utilizando una media mÃ³vil. Sin embargo, la calidad de las imputaciones puede no ser buena, lo que perjudica el entrenamiento del modelo. Una forma de mejorar la estrategia de imputaciÃ³n es combinarla con weighted time series forecasting. Esta Ãºltima consiste en reducir el peso de las observaciones imputadas y, por tanto, su influencia durante el entrenamiento del modelo.

Este documento muestra dos ejemplos de cÃ³mo skforecast facilita la aplicaciÃ³n de esta estrategia.

Dar peso de cero a los valores imputados¶

Para minimizar la influencia en el modelo de los valores imputados, se define una funciÃ³n que crea pesos acorde a las siguientes reglas:

  • Peso de 0 si la fecha del Ã­ndice al periodo imputado o estÃ¡ a mÃ©nos de 14 dÃ­as de Ã©l.

  • Peso de 1 en caso contrario.

Si una observaciÃ³n tiene un peso de 0, no influye en el entrenamiento del modelo.

✎ Nota

Los valores imputados no deben participar en el proceso de entrenamiento ni como *target* ni como predictor (lags). Por lo tanto, tambiÃ©n deben excluirse los valores dentro de una ventana de tamaÃ±o tan grande como los lags utilizados.

# FunciÃ³n para asignar pesos a las observaciones
# ==============================================================================
def custom_weights(index):
    """
    Devuelve 0 si el Ã­ndice se encuentra en algÃºn gap.
    """
    gaps = [
        ['2020-09-01', '2020-10-10'],
        ['2020-11-08', '2020-12-15'],
    ]
    
    missing_dates = [pd.date_range(
                        start = pd.to_datetime(gap[0]) + pd.Timedelta('14d'),
                        end   = pd.to_datetime(gap[1]) + pd.Timedelta('14d'),
                        freq  = 'D'
                    ) for gap in gaps]
    missing_dates = pd.DatetimeIndex(np.concatenate(missing_dates))   
    weights = np.where(index.isin(missing_dates), 0, 1)

    return weights

Se entrena de nuevo un ForecasterAutoreg pero esta vez incluyendo la funciÃ³n custom_weights.
