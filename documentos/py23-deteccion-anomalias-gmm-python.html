Introducción¶


Gaussian Mixture Model (GMM) es un modelo probabilstico en el que se considera que las observaciones siguen una distribucin probabilstica formada por la combinacin de mltiples distribuciones normales (componentes). Puede entenderse como una generalizacin de K-means con la que, en lugar de asignar cada observacin a un nico cluster, se obtiene una distribucin probabilidad de pertenencia a cada uno.

Ajustar un modelo GMM consiste en estimar los parmetros que definen la funcin de distribucin de cada componente: la media y la matriz de covarianza. Por ejemplo, si el modelo tiene dos componentes, hay que encontrar la media y la matriz de covarianzas de cada una. Si es un problema multidimensional, por ejemplo de 3 variables, la media de cada componente es un vector de 3 valores y la matriz de covarianza una matriz de 3x3. El algoritmo ms empleado para realizar el ajuste es Expectation-Maximization (EM).

Una vez aprendidos los parmetros, se puede calcular la probabilidad que tiene cada observacin de pertenecer a cada componente y al conjunto de la distribucin. Observaciones con muy poca probabilidad pueden considerarse como anomalas.

Aspectos prcticos

Al entrenar un modelo GMM, junto con el nmero de componentes, hay que determinar el tipo de matriz de covarianza. Dependiendo esto, la forma de las distribuciones de las componentes puede ser:

  • tied: todas las componentes comparten la misma matriz de covarianza.

  • diagonal: las dimensiones de cada componente a lo largo de cada dimensin puede ser distintas, pero siempre quedan alineadas con los ejes, es decir, su orientaciones son limitadas.

  • spherical: las dimensiones de cada componente son las mismas en todas las dimensiones. Esto permite generar distribuciones de distinto tamao pero todas esfricas.

  • full: cada componente tiene su propia matriz de covarianza, por lo que pueden tener cualquier orientacin y dimensiones.

Ejemplo 1¶

Modelo¶


Con la clase sklearn.mixture.GaussianMixture de Scikit-Learn se pueden entrenar modelos GMMs utilizando el algoritmo expectation-maximization (EM) . Entre sus parmetros destacan:

  • n_components: nmero de componentes (en este caso clusters) que forman el modelo.

  • covariance_type: tipo de matriz de covarianza (full (default), tied, diag, spherical).

  • max_iter: nmero mximo de iteraciones permitidas.

  • random_state: semilla para garantizar la reproducibilidad de los resultados.

In [25]:
# Definicin y entrenamiento del modelo GMM
# ==============================================================================
modelo_gmm = GaussianMixture(
                n_components    = 4,
                covariance_type = 'full',
                random_state    = 123
             )

modelo_gmm.fit(X=datos_X)
Out[25]:
GaussianMixture(n_components=4, random_state=123)

El objeto devuelto por GaussianMixture contiene entre otros datos: el peso de cada componente (cluster) en el modelo (weights_), su media (means_) y matriz de covarianza (covariances_). La estructura de esta ltima depende del tipo de matriz empleada en el ajuste del modelo (covariance_type).

Cmo saber el nmero de componentes y tipo de matriz de covarianza?

Al tratarse de un problema no supervisado, no hay forma de conocer de antemano el nmero de componentes y tipo de matriz de covarianza ptimos. Afortunadamente, al ser un modelo probabilstico, se puede recurrir a mtricas como el Akaike information criterion (AIC) o Bayesian information criterion (BIC) para identificar cmo de bien se ajustan los datos observados al modelo creado, a la vez que se controla el exceso de overfitting. En la implementacin de Scikit Learn, para ambas mtricas, cuanto ms bajo el valor, mejor.

# Tunning del modelo GMM
# ==============================================================================
fig, ax = plt.subplots(figsize=(6, 3.84))

n_components = range(1, 40)
covariance_types = ['spherical', 'tied', 'diag', 'full']

    valores_bic = []
    
    for i in n_components:
        modelo = GaussianMixture(n_components=i, covariance_type=covariance_type)
        modelo = modelo.fit(datos_X)
        valores_bic.append(modelo.bic(datos_X))
        
    ax.plot(n_components, valores_bic, label=covariance_type)
        
ax.set_title("Valores BIC")
ax.set_xlabel("Nmero componentes")
ax.legend();

Detección de anomalías¶


Una vez calculadas la probabilidad de cada observacin acorde al modelo, se pueden emplear como criterio para identificar anomalas. Cabra esperar que, observaciones con una probabilidad predicha muy baja, sean anmalas.

En la prctica, si se est empleando esta estrategia de deteccin es porque no se dispone de datos etiquetados, es decir, no se conoce qu observaciones son realmente anomalas. Sin embargo, como en este ejemplo se dispone de la clasificacin real, se puede verificar si realmente los datos anmalos tienen probabilidades menores.

La distribucin del logaritmo de probabilidad en el grupo de las anomalas es inferior. Sin embargo, al existir un solapamiento notable, si se clasifican las n observaciones con menor probabilidad como anomalas, se incurrira en errores de falsos positivos.

Acorde a la documentacin, el set de datos Cardiotocogrpahy contiene 176 anomalas. Vase la matriz de confusin resultante si se clasifican como anomalas las 176 observaciones con menor probabilidad predicha.

De las 176 observaciones identificadas como anomalas, solo el 32% lo son realmente. El porcentaje de falsos positivos es muy elevado, el mtodo de GMM no consigue resultados notables en este set de datos.

Una de las principales limitaciones del uso de modelos GMM como detectores de anomalas es que consideran que los datos siguen distribuciones normales multivariante. En la prctica, esta asuncin es difcil que se cumpla, sobretodo a medida que aumenta el nmero de variables. Una estrategia que en ocasiones consigue mitigar parte de este problema es reducir la dimensionalidad de los datos con PCA y luego aplicar un modelo GMM con solo unas pocas componentes.

Reentrenamiento iterativo¶


El modelo anterior se ha entrenado empleando todas las observaciones, incluyendo potenciales anomalas. Dado que el objetivo es aprender las distribuciones de las componentes nicamente con los datos normales, se puede mejorar el resultado reentrenando el modelo pero excluyendo las n observaciones con menor probabilidad (potenciales anomalas).

Se repite la deteccin de anomalas pero, esta vez, descartando las observaciones con una probabilidad inferior al cuantil 0.01.
