Modelos de forecasting globales¶

En los modelos de series individuales (Local Forecasting Model), una Ãºnica serie temporal se modela como una combinaciÃ³n lineal o no lineal de sus valores pasados y, opcionalmente, variables exÃ³genas. Aunque este mÃ©todo proporciona una comprensiÃ³n exhaustiva de cada serie, su escalabilidad puede verse comprometida cuando se dispone de un gran nÃºmero de series.

Los modelos globales multiserie (Global Forecasting Model) consisten en un Ãºnico modelo que tiene en cuenta varias series temporales de forma simultÃ¡nea. Intentan captar los patrones bÃ¡sicos que rigen las series, mitigando asÃ­ el ruido potencial que pueda introducir cada serie. Este enfoque es eficiente desde el punto de vista computacional, es fÃ¡cil de mantener y puede conseguir mayor generalizaciÃ³n, aunque potencialmente a costa de sacrificar cierta resoluciÃ³n individual. Se pueden distinguir dos estrategias de modelos de previsiÃ³n global: MÃºltiples series temporales independientes y MÃºltiples series temporales dependientes.

Ventajas de los modelos multiseries

  • Es mÃ¡s fÃ¡cil mantener y controlar un solo modelo que varios.

  • Dado que todas las series temporales se combinan durante el entrenamiento, cuando las series sean cortas (pocos datos) el modelo tendrÃ¡ una mayor capacidad de aprendizaje al disponer de mÃ¡s observaciones.

  • Al combinar mÃºltiples series temporales, el modelo puede aprender patrones mÃ¡s generalizables.

Desventajas de los modelos multiseries

  • Si las series no siguen la misma dinÃ¡mica interna, el modelo puede aprender un patrÃ³n que no represente a ninguna de ellas.

  • Las series pueden enmascararse unas a otras, por lo que el modelo puede no predecirlas todas con el mismo rendimiento.

  • Es mÃ¡s exigente desde el punto de vista computacional (tiempo y recursos) entrenar y realizar backtesting de un modelo grande que de varios pequeÃ±os.

MÃºltiples series temporales independientes

En esta situaciÃ³n, cada serie temporal es independiente de las demÃ¡s o, dicho de otro modo, los valores pasados de una serie no se utilizan como predictores de las otras series. Â¿Por quÃ© es Ãºtil entonces modelar todo junto? Aunque las series no dependen unas de otras, pueden seguir el mismo patrÃ³n intrÃ­nseco en cuanto a sus valores pasados y futuros. Por ejemplo, en una misma tienda, las ventas de los productos A y B pueden no estar relacionadas, pero siguen la misma dinÃ¡mica, la de la tienda.

Para predecir los siguientes n steps, se sigue una estrategia recurisva, recursive multi-step forecasting.

MÃºltiples series temporales dependientes

Todas las series se modelan teniendo en cuenta que cada serie temporal depende no sÃ³lo de sus valores pasados, sino tambiÃ©n de los valores pasados de las demÃ¡s series. Se espera que el modelo no sÃ³lo aprenda la informaciÃ³n de cada serie por separado, sino que tambiÃ©n las relacione. Por ejemplo, las mediciones realizadas por todos los sensores (caudal, temperatura, presiÃ³n...) instalados en una mÃ¡quina industrial como un compresor. Series temporales multivariantes user guide.

En este primer ejemplo, se utilizan mÃºltiples series temporales de la misma longitud. El objetivo es comparar los resultados de un modelo global con los de un modelo individual para cada serie al predecir los prÃ³ximos 7 dÃ­as de ventas para 50 productos diferentes en una tienda, utilizando los 5 aÃ±os de historial disponible. Los datos se han obtenido del Store Item Demand Forecasting Challenge. Este conjunto de datos contiene 913,000 transacciones de ventas desde el 01/01/2013 hasta el 31/12/2017 para 50 productos (SKU) en 10 tiendas.

Forecasting individual para cada item¶

Se entrena un modelo Gradient Boosting Machine (GBM) para cada producto, utilizando las ventas de los Ãºltimos 14 dÃ­as, asÃ­ como las ventas promedio, mÃ¡xima y mÃ­nima de los Ãºltimos 7 dÃ­as como predictores. El rendimiento del modelo para los siguientes 7 dÃ­as se evalÃºa mediante backtesting utilizando el Error Absoluto Medio (MAE) como mÃ©trica de evaluaciÃ³n. Finalmente, se compara el rendimiento de estos modelos individuales con el de un modelo global entrenado con todas las series.

# Entrenar y realizar backtesting de un modelo para cada item
# ======================================================================================
items = []
mae_values = []
predictions = {}

for i, item in enumerate(tqdm(data.columns)):
    # Definir el forecaster
    window_features = RollingFeatures(stats=['mean', 'min', 'max'], window_sizes=7)
    forecaster = ForecasterRecursive(
                     regressor       = HistGradientBoostingRegressor(random_state=8523),
                     lags            = 14,
                     window_features = window_features
                 )
    # Backtesting forecaster
    cv = TimeSeriesFold(
            steps              = 7,
            initial_train_size = len(data_train) + len(data_val),
            refit              = False,
         )
    metric, preds = backtesting_forecaster(
                        forecaster    = forecaster,
                        y             = data[item],
                        cv            = cv,
                        metric        = 'mean_absolute_error',
                        verbose       = False,
                        show_progress = False
                    )
    items.append(item)
    mae_values.append(metric.at[0, 'mean_absolute_error'])
    predictions[item] = preds

# Resultados
uni_series_mae = pd.Series(
                     data  = mae_values,
                     index = items,
                     name  = 'uni_series_mae'
                 )

Series de diferente longitud y distintas variables exógenas¶














En escenarios en los que se tienen que modelar mÃºltiples series, es habitual que las series tengan longitudes distintas debido a diferencias en los momentos de inicio del registro de los datos. Para hacer frente a este escenario, la clase ForecasterAutoregMultiSeries permite modelizar simultÃ¡neamente series temporales de distintas longitudes y con distintas variables exÃ³genas.


    

  • Cuando las series modeladas tienen longitudes diferentes, deben almacenarse en un diccionario Python. Las claves del diccionario son los nombres de las series y los valores son las propias series. Todas las series deben ser del tipo pandas.Series, tener un Ã­ndice datetime y tener la misma frecuencia.

Los datos de este ejemplo estÃ¡n almacenados en "formato largo" en un Ãºnico DataFrame. La columna series_id identifica la serie a la que pertenece cada observaciÃ³n. Optimización de hiperparámetros (tuning) y selección de lags¶














En el primer ejemplo de este documento, la comparaciÃ³n entre forecaster se ha realizado sin optimizar los hiperparÃ¡metros de los regresores. Para una comparaciÃ³n justa, se utiliza una estrategia de grid search con el fin de seleccionar la mejor configuraciÃ³n para cada forecaster. VÃ©ase mÃ¡s informaciÃ³n en hyperparameter tuning and lags selection.



















    
    ð Note



Optimizar los hiperparÃ¡metros de mÃºltiples modelos puede ser computacionalmente costoso. Para acelerar el proceso, la evaluaciÃ³n de cada configuraciÃ³n candidata se realiza utilizando *one-step-ahead* en lugar de *backtesting*. Para mÃ¡s detalles sobre las ventajas y limitaciones de este enfoque, consulte [One-step-ahead validation](https://skforecast.org/latest/user_guides/hyperparameter-tuning-and-lags-selection#one-step-ahead-validation.html).

Selección de predictores¶














La selecciÃ³n de predictores es el proceso de seleccionar un subconjunto de predictores relevantes (variables) para su uso en la construcciÃ³n del modelo. Las tÃ©cnicas de selecciÃ³n de predictores se utilizan por varias razones: para simplificar los modelos y hacerlos mÃ¡s fÃ¡ciles de interpretar, para reducir el tiempo de entrenamiento, para evitar los problemas de dimensionalidad, para mejorar la generalizaciÃ³n reduciendo el sobreajuste (formalmente, la reducciÃ³n de la varianza), entre otros.


Skforecast es compatible con los mÃ©todos de selecciÃ³n implementados en scikit-learn. Existen varios mÃ©todos de selecciÃ³n de caracterÃ­sticas, pero los mÃ¡s comunes son:


    

  • Recursive feature elimination (RFE)
    

    • 

  • Sequential Feature Selection (SFS)
    

    • 

  • Feature selection based on threshold (SelectFromModel)
    

    • 




















    
    💡 Tip



La selecciÃ³n de predictores es una herramienta poderosa para mejorar el rendimiento de los modelos de machine learning. Sin embargo, es computacionalmente costosa y puede llevar tiempo. Dado que el objetivo es encontrar el mejor subconjunto de variables, no el mejor modelo, no es necesario utilizar todo el conjunto de datos o un modelo muy complejo. En su lugar, se recomienda utilizar un pequeÃ±o subconjunto de datos y un modelo simple. Una vez que se haya identificado el mejor subconjunto de variables, el modelo puede entrenarse utilizando todo el conjunto de datos y una configuraciÃ³n mÃ¡s compleja.
















Pesos en forecasting multiseries¶














Los pesos se utilizan para controlar la influencia que tiene cada observaciÃ³n en el entrenamiento del modelo. ForecasterAutoregMultiseries acepta dos tipos de pesos:


    

  • series_weights controla la importancia relativa de cada serie. Si una serie tiene el doble de peso que las demÃ¡s, las observaciones de esa serie influyen el doble en el entrenamiento. Cuanto mayor sea el peso de una serie en relaciÃ³n con las demÃ¡s, mÃ¡s se centrarÃ¡ el modelo en intentar aprender esa serie.
    

    • 

  • weight_func controla la importancia relativa de cada observaciÃ³n en funciÃ³n del Ã­ndice. Por ejemplo, una funciÃ³n que asigna un peso menor a ciertas fechas.
    

    • 



Si se indican los dos tipos de pesos, estos se multiplican para crear los pesos finales como se muestra en la figura. El sample_weight resultante no puede contener valores negativos.


MÃ¡s informaciÃ³n sobre weights in multi-series forecasting y weighted time series forecasting con skforecast.


En este ejemplo, item_1 tiene una mayor importancia relativa entre series (pesa 3 veces mÃ¡s que el resto de series), y las observaciones entre '2013-12-01' y '2014-01-31' se consideran no representativas y se les aplica un peso de 0.
