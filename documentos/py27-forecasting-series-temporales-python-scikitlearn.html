Introducción¶


Una serie temporal (time series) es una sucesiÃ³n de datos ordenados cronolÃ³gicamente, espaciados a intervalos iguales o desiguales. El proceso de Forecasting consiste en predecir el valor futuro de una serie temporal, bien modelando la serie temporal Ãºnicamente en funciÃ³n de su comportamiento pasado (autorregresivo) o empleando otras variables externas a la serie temporal.

A lo largo de este documento, se describe cÃ³mo utilizar modelos de regresiÃ³n de Scikit-learn para realizar forecasting sobre series temporales. En concreto, se hace uso de Skforecast, una librerÃ­a sencilla (en construcciÃ³n) que contiene las clases y funciones necesarias para adaptar cualquier modelo de regresÃ³n de Scikit-learn a problemas de forecasting.

Multi-Step Time Series Forecasting¶


Cuando se trabaja con series temporales, raramente se quiere predecir solo el siguiente elemento de la serie ($t_{+1}$), sino todo un intervalo futuro o un punto alejado en el tiempo ($t_{+n}$). A cada paso de predicciÃ³n se le conoce como step.

Existen varias estrategias que permiten generar este tipo de predicciones mÃºltiples.

Recursive multi-step forecasting

Dado que, para predecir el momento $t_{n}$ se necesita el valor de $t_{n-1}$, y $t_{n-1}$ se desconoce, es necesario hacer predicciones recursivas en las que, cada nueva predicciÃ³n, se basa en la predicciÃ³n anterior. A este proceso se le conoce como recursive forecasting o recursive multi-step forecasting.

La principal adaptaciÃ³n que se necesita hacer para aplicar modelos de scikit learn a problemas de recursive multi-step forecasting es transformar la serie temporal en un matriz en la que, cada valor, estÃ¡ asociado a la ventana temporal (lags) que le preceden. Esta estrategia de forecasting pueden generarse fÃ¡cilmente con las clases ForecasterAutoreg y ForecasterCustom de la librerÃ­a skforecast.

TranformaciÃ³n de una serie temporal en una matriz de 5 lags y un vector con el valor de la serie que sigue a cada fila de la matriz.

Direct multi-step forecasting

Esta estrategia consiste en entrenar un modelo distinto para cada step. Por ejemplo, si se quieren predecir los siguientes 5 valores de una serie temporal, se entrenan 5 modelos distintos, uno para cada step. Como resultado, las predicciones son independientes unas de otras. Esta estrategia de forecasting pueden generarse fÃ¡cilmente con la clase ForecasterAutoregMultiOutput de la librerÃ­a skforecast.

Multiple output forecasting

Determinados modelos son capaces de predecir de forma simultÃ¡nea varios valores de una secuencia (one-shot). Un ejemplo de modelo con esta capacidad son las redes neuronales LSTM.

Forecasting autorregresivo recursivo¶


Se dispone de una serie temporal con el gasto mensual (millones de dÃ³lares) en fÃ¡rmacos con corticoides que tuvo el sistema de salud Australiano entre 1991 y 2008. Se pretende crear un modelo autoregresivo capaz de predecir el futuro gasto mensual. Para poder predecir mÃºltiples valores a futuro, se emplea la estrategia recursiva (recursive multi-step forecasting).

Importancia predictores¶


Dado que el objeto ForecasterAutoreg utiliza modelos scikit-learn, una vez entrenado, se puede acceder a la importancia de los predictores. La librerÃ­a statsmodels dispone de algunos de los principales modelos de forecasting (AutoReg, AR, ARMA, ARIMA). Se verifica que se obtienen los mismos resultados con AutoReg que con la clase ForecasterAutoreg si se utiliza un modelo de regresor lineal LinearRegression().

Forecasting autorregresivo recursivo con predictores custom¶


En determinados escenarios, puede ser interesante incorporar otras caracterÃ­sticas de la serie temporal ademÃ¡s de los lags, por ejemplo, la media movil de los Ãºltimos n valores puede servir para capturar la tendencia de la serie.

La clase ForecasterCustom se comporta de forma muy similar a la clase ForecasterAutoreg vista en los apartados anteriores pero con la diferencia de que, es el usuario, quien define la funciÃ³n empleada para crear los predictores.

Se repite el primer ejemplo del documento, predecir los Ãºltimos 36 meses de la serie temporal, pero esta vez, utilizando como predictores los 10 primeros lags y la media mÃ³vil de los Ãºltimos 20 meses.

ForecasterCustom¶


Se crea y entrena ForecasterCustom a partir de un regresor RandomForestRegressor. Para crear los predictores, se emplea la funciÃ³n create_predictors() que calcula los primeros 10 lags y la media mÃ³vil de los Ãºltimos 20 valores.

In [50]:
# FunciÃ³n para calcular los predictores a partir de la serie temporal
# ==============================================================================
def create_predictors(y):
    '''
    Crear los primeros 10 lags.
    Calcular la media mÃ³vil de los Ãºltimos 20 valores.
    '''
    
    X_train = pd.DataFrame({'y':y.copy()})
    for i in range(0, 10):
        X_train[f'lag_{i+1}'] = X_train['y'].shift(i)
        
    X_train['moving_avg'] = X_train['y'].rolling(20).mean()
    
    X_train = X_train.drop(columns='y').tail(1).to_numpy()  
    
    return X_train

Al crear el forecaster, el argumento window_size debe ser un valor como mÃ­nimo tan grande como la ventana que utiliza la funciÃ³n que crea los predictores. En este caso, 20.

Direct multi-step forecasting¶


Los modelos ForecasterAutoreg y ForecasterCustom siguen una estrategia de predicciÃ³n recursiva en la que, cada nueva predicciÃ³n, se basa en la predicciÃ³n anterior. Una alternativa es entrenar un modelo para cada uno de los steps que se desea predecir. Esta estrategia, normalmente conocida como direct multi-step forecasting, es computacionalmente mÃ¡s costosa que la recursiva puesto que requiere entrenar mÃºltiples modelos. Sin embargo, en algunos escenarios, consigue mejores resultados. Este tipo de modelos pueden obtenerse con la clase ForecasterAutoregMultiOutput y pueden incluir tambiÃ©n una o mÃºltiples variables exÃ³genas.

Intervalos de predicción¶


Un intervalo de predicciÃ³n define el intervalo dentro del cual es de esperar que se encuentre el verdadero valor de $y$ con una determinada probabilidad.

Rob J Hyndman y George Athanasopoulos, listan en su libro Forecasting: Principles and Practice mÃºtiples formas de estimar intervalos de predicciÃ³n, la mayorÃ­a los cuales requieren que los resudios (errores) del modelo se distribuyan de forma normal. Cuando no se puede asumir esta propiedad, se puede recurrir a bootstrapping, que solo asume que los residuos no estÃ¡n correlacionados. Este es el mÃ©todo utilizado en la librerÃ­a skforecast para los modelos de tipo ForecasterAutoreg y ForecasterCustom.
